Setting the service's type to "own" instead of "shared" causes it to run in it's own process even while it remains in the same named group. It changes that service's Type value in HKLM/SYSTEM/CurrentControlSet/Services/@name@ to 16 instead of 32, causing the service to run in it's own host process. You don't have to do it manually in the registry, you can just run: sc.exe config [servicename] . You should run sc.exe config by itself to see the help, because it enumerates the options for type, and also lets you set services to start delayed, etc or change their group. After you make this change, you have to stop and start the service to get it to run in it's own host process. You can do that from the command-line using sc.exe stop [servicename] and sc.exe start [servicename]

To enable ActiveX settings:
1. Open Internet Explorer.
2. Click the Tools menu, then click Internet Options....
3. Click the Security tab.
4. Click the Internet icon, then click Default level to restore the Security level to Medium.
5. Click OK.
OR
Internet Explorer as Admin> Internet Options> Security> Custom> Enable ActiveX functions.
